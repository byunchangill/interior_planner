# HomeStyler 모듈 로드맵 (P1 범위)

v1 범위는 P1 핵심 흐름이다. 관리자 백오피스(ADM), 위시리스트(FR-SAVE-005), 데이터 다운로드(FR-DATA-005)는 범위 외 — 요청 시 하네스 확장으로 처리.

## 모듈 순서와 의존성

M0 → M1 → M2 → M3 → M4 → M5 순서를 권장한다. M2(공간) 없이 M3(추천)를 만들 수 없고, M3 없이 M4(저장·공유)를 만들 수 없다.

### M0: 스캐폴딩 (의존성 없음)

| 항목 | 내용 |
|------|------|
| 백엔드 | Spring Boot 3.x Gradle 프로젝트, 공통 응답 포맷(`ApiResponse`), 전역 예외 핸들러, H2/PostgreSQL 프로파일, CORS 설정 |
| 프론트엔드 | Vite + React + TS + Tailwind, 디자인 토큰 이식(DESIGN.md frontmatter → tailwind.config), 라우터 셸, 하단 탭 네비게이션(홈/내 공간/추천/보관함/마이), axios 인스턴스 + 토큰 인터셉터 |
| QA | BE 기동 + FE 빌드 + `/api/v1/health` 연동 확인 |

### M1: AUTH — 인증/계정

| 항목 | 내용 |
|------|------|
| FR | FR-AUTH-002(이메일 가입/로그인), FR-AUTH-003(이미지 데이터 동의), FR-AUTH-005(자동 로그인/토큰 갱신). 소셜(FR-AUTH-001)은 키 확보 전까지 버튼만 배치 |
| 화면 | COM-001 스플래시(_27), COM-002 온보딩(_31, _32), COM-003 로그인(_21), COM-004 약관 동의(_20), COM-005 권한 안내(_33) |
| API | POST /auth/signup, /auth/login, /auth/refresh, /auth/consents |

### M2: SPACE — 공간 등록

| 항목 | 내용 |
|------|------|
| FR | FR-SPACE-001~008 |
| 화면 | SPACE-001 목록(_18), SPACE-002 유형 선택(_2, _24), SPACE-003 촬영 가이드(_3), SPACE-004 도면 업로드(_22), SPACE-005 치수(_9), SPACE-006 가구(_9), SPACE-007 상세(_23) |

| 주의 | 치수·가구 "AI 추정"은 Mock(업로드 시 그럴듯한 기본값 생성). 웹 환경이므로 카메라 촬영은 `<input capture>` 기반 |

### M3: RECO — 조건 설정·AI 분석·추천 결과

| 항목 | 내용 |
|------|------|
| FR | FR-PREF-001~004, FR-RECO-001~012 |
| 화면 | RECO-001/002 조건 설정(_10), RECO-003 분석 진행(ai_1, ai_2), RECO-004 요약(_4), RECO-005 상세, RECO-006 전후 비교(_5), RECO-007 가구 상세(_17), RECO-008 전문가 확인(_11) |

| 주의 | 분석은 비동기 필수(@Async + 상태 폴링). Mock AI는 단계별 진행률(구조→채광→동선→생성)을 시뮬레이션. 모든 결과 화면에 AI 한계 고지(FR-RECO-012) 포함 |

### 참고 화면 (모듈 외)

HOME-001 홈(_1), HOME-002 스타일 갤러리(_15), HOME-003 스타일 상세(_7, _19)는 M0 이후 아무 때나 구현 가능한 독립 화면(정적 콘텐츠 + API). M1과 함께 처리하는 것을 권장.
