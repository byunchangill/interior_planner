# M2 계약 — SPACE (공간 등록·사진·치수·가구)

공통 규칙은 `api-contract-guide.md` 표준을 따른다 (Base URL `/api/v1`, ApiResponse 포맷, camelCase, `Authorization: Bearer {accessToken}`). 모든 엔드포인트 **인증 필요**, 소유자 본인만 접근(타인 리소스 → AUTH_003).

## 범위 및 결정사항

- FR-SPACE-001~008. AI 추정(치수·가구 인식)은 **Mock** — 업로드 시 그럴듯한 기본값 생성
- 화면: SPACE-001 목록(_18), SPACE-002 유형선택(_24), SPACE-003 촬영가이드(_3), SPACE-004 도면업로드(_22), SPACE-005 치수(_9), SPACE-006 가구(_9), SPACE-007 상세(_23)
- **결정 1 (집/Home 엔티티 생략)**: 명세서의 `homeId`는 v1에서 도입하지 않는다. 공간(Space)은 사용자에 직접 소속. 다중 집 관리는 범위 밖(필요 시 후속) — 명세서 `homeId` 파라미터는 계약에서 제거
- **결정 2 (업로드 방식)**: 명세서의 서명 URL 직접 업로드 흐름 대신, dev는 **백엔드로 직접 multipart 업로드**(로컬 파일시스템 저장, 저장 경로 추상화). 서명 URL/오브젝트 스토리지는 prod 전환 시 어댑터 교체
- 업로드 필수 처리: 확장자·MIME 검증, **EXIF 메타데이터 제거(GPS)**, 20MB 제한, UUID 파일명, 응답엔 `/files/**` 서빙 URL만 노출
- ID 타입: 명세서 예시는 `sp_123` 문자열이나, M1과 일관되게 **Long 자동증가**를 사용(응답 필드는 그대로 `spaceId` 등)

## 엔드포인트

### POST /spaces

- **기능**: FR-SPACE-001 공간 생성(유형 선택 직후)
- **요청**: `{ "spaceType": "LIVING_ROOM", "name": "거실" }` (name nullable — 미입력 시 유형 한글명 기본값)
- **응답 200**:
  ```json
  { "success": true, "data": { "spaceId": 1, "spaceType": "LIVING_ROOM", "name": "거실", "photoCount": 0, "createdAt": "2026-07-20T10:00:00Z" } }
  ```
- **에러**: VALID_003(정의되지 않은 spaceType)

### GET /spaces

- **기능**: FR-SPACE-007 내 공간 목록 (SPACE-001)
- **응답 200**:
  ```json
  { "success": true, "data": { "items": [
    { "spaceId": 1, "spaceType": "LIVING_ROOM", "name": "거실", "photoCount": 3, "thumbnailUrl": "/files/ab12.jpg", "createdAt": "2026-07-20T10:00:00Z" }
  ] } }
  ```
  - 사진 없으면 `thumbnailUrl: null`
- **에러**: 없음(빈 목록 `items: []`)

### GET /spaces/{spaceId}

- **기능**: FR-SPACE-007 공간 상세 (SPACE-007) — 사진·치수·가구 전체
- **응답 200**:
  ```json
  { "success": true, "data": {
    "spaceId": 1, "spaceType": "LIVING_ROOM", "name": "거실",
    "photos": [ { "photoId": 10, "url": "/files/ab12.jpg", "isFloorPlan": false } ],
    "dimensions": { "widthM": 4.2, "depthM": 3.5, "heightM": 2.3, "areaPyeong": 4.4,
                    "confidence": "MEDIUM", "isUserVerified": false,
                    "openings": [ { "openingId": 1, "type": "WINDOW", "wall": "NORTH", "widthM": 1.8 } ] },
    "furniture": [ { "furnitureId": 20, "type": "SOFA", "label": "3인용 소파", "keep": false, "source": "AI_DETECTED" } ]
  } }
  ```
  - `dimensions` nullable(사진 등록 전 null), `openings`/`furniture` 빈 배열 가능
- **에러**: RES_001(없음), AUTH_003(타인 소유)

### DELETE /spaces/{spaceId}

- **기능**: FR-SPACE-007 공간 삭제 — 사진 파일 물리 삭제 포함
- **응답 200**: `{ "success": true, "data": { "spaceId": 1 } }`
- **에러**: RES_001, AUTH_003

### POST /spaces/{spaceId}/photos

- **기능**: FR-SPACE-002/003 사진 등록(촬영·사진첩 공통, 웹은 `<input>` 기반). 업로드 시 Mock AI가 치수·가구를 추정해 최초 1회 채움
- **요청**: `multipart/form-data` — `file` (JPG/PNG, 최대 20MB), `isFloorPlan` (boolean, 기본 false; 도면 업로드 FR-SPACE-004는 true)
- **응답 200**:
  ```json
  { "success": true, "data": {
    "photoId": 10, "url": "/files/ab12.jpg", "isFloorPlan": false,
    "qualityCheck": "PASSED",
    "detectedFurniture": [ { "furnitureId": 20, "type": "SOFA", "label": "3인용 소파" } ]
  } }
  ```
  - `qualityCheck`: PASSED | LOW_LIGHT | BLURRY | NOT_RECOGNIZED (Mock은 항상 PASSED, FR-SPACE-008은 스텁)
  - `detectedFurniture`: 최초 사진에서만 Mock 생성, 이후 빈 배열
- **에러**: VALID_002(형식/용량), VALID_003(장수 초과 — 공간당 10장), IMG_001(품질 미달 — Mock에선 미발생)

### DELETE /spaces/{spaceId}/photos/{photoId}

- **기능**: 사진 삭제(파일 물리 삭제)
- **응답 200**: `{ "success": true, "data": { "photoId": 10 } }`
- **에러**: RES_001, AUTH_003

### PATCH /spaces/{spaceId}/dimensions

- **기능**: FR-SPACE-005 치수 확인·수정. 사용자 수정 시 `isUserVerified: true` 저장
- **요청**:
  ```json
  { "widthM": 4.5, "depthM": 3.6, "heightM": 2.3, "isUserVerified": true,
    "openings": [ { "type": "WINDOW", "wall": "NORTH", "widthM": 1.8 } ] }
  ```
  - widthM/depthM: 1.0~20.0, heightM: 2.0~5.0(nullable, 기본 2.3), 소수 첫째 자리
  - openings nullable(미전송 시 기존 유지)
- **응답 200**:
  ```json
  { "success": true, "data": { "spaceId": 1, "widthM": 4.5, "depthM": 3.6, "heightM": 2.3,
    "areaPyeong": 4.9, "isUserVerified": true,
    "openings": [ { "openingId": 1, "type": "WINDOW", "wall": "NORTH", "widthM": 1.8 } ] } }
  ```
- **에러**: VALID_003(범위 초과), RES_001, AUTH_003

### PUT /spaces/{spaceId}/furniture

- **기능**: FR-SPACE-006 기존 가구 목록 갱신(유지 선택 + 수동 추가/수정/제거를 전체 목록 교체로 처리)
- **요청**:
  ```json
  { "furniture": [
    { "furnitureId": 20, "type": "SOFA", "label": "3인용 소파", "keep": true },
    { "type": "TABLE", "label": "4인 식탁", "keep": false }
  ] }
  ```
  - `furnitureId` 있으면 수정, 없으면 신규 추가(source=USER_ADDED). 요청에 빠진 기존 항목은 삭제
- **응답 200**:
  ```json
  { "success": true, "data": { "furniture": [
    { "furnitureId": 20, "type": "SOFA", "label": "3인용 소파", "keep": true, "source": "AI_DETECTED" },
    { "furnitureId": 21, "type": "TABLE", "label": "4인 식탁", "keep": false, "source": "USER_ADDED" }
  ] } }
  ```
- **에러**: VALID_003(정의되지 않은 type), RES_001, AUTH_003

## Enums

- SpaceType: LIVING_ROOM | BEDROOM | KITCHEN | STUDY | ETC
- FurnitureType: SOFA | BED | TABLE | CHAIR | WARDROBE | BOOKSHELF | TV_STAND | DRAWER | LIGHTING | ETC
- OpeningType: WINDOW | DOOR
- Wall: NORTH | SOUTH | EAST | WEST
- Confidence: HIGH | MEDIUM | LOW
- FurnitureSource: AI_DETECTED | USER_ADDED
- QualityCheck: PASSED | LOW_LIGHT | BLURRY | NOT_RECOGNIZED

## 에러 코드

| 코드 | 의미 |
|------|------|
| AUTH_003 | 접근 권한 없음(타인 리소스) |
| RES_001 | 리소스 없음 |
| VALID_002 | 파일 형식/용량 오류 |
| VALID_003 | 입력값 범위/enum 오류(치수 범위, 장수 초과, 미정의 enum) |
| IMG_001 | 이미지 품질 미달(Mock 미발생, FR-SPACE-008 스텁) |

## Mock AI 동작 (backend AiEstimationService)

- 사진 최초 등록 시: 공간 유형별 그럴듯한 치수(예: LIVING_ROOM 4.2×3.5×2.3, confidence=MEDIUM)와 대표 가구 1~2종(SOFA 등) 생성
- 인터페이스로 분리해 M3 실제 AI 어댑터 교체 가능하게 유지
