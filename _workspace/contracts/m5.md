# M5 계약 — MY/DATA (마이페이지·데이터 관리·탈퇴)

공통 규칙은 `api-contract-guide.md` 표준을 따른다. 모든 엔드포인트 인증 필요, 본인 데이터만.

## 범위 및 결정사항

- FR-DATA-001(원본 조회), FR-DATA-002(원본 삭제·물리), FR-DATA-003(삭제 후 결과 유지), FR-DATA-004(탈퇴 전체 삭제)
- 화면: MY-001 마이페이지(_8), MY-002 데이터·개인정보 관리(_12), MY-003 계정 설정(_14), MY-004 회원 탈퇴(_25), MY-005 공지·고객센터(_26, P2 — **화면만**, API 없음)
- **결정 1 (삭제는 물리 삭제)**: 원본 사진·도면은 스토리지 파일까지 즉시 물리 삭제(M2 `FileStorageService.delete`). soft delete 아님. 백업본 30일 파기는 dev 스토리지에 백업 개념이 없으므로 범위 밖(주석만).
- **결정 2 (탈퇴 확인)**: 회원 탈퇴는 비밀번호 재확인 필수(요청 본문 `password`). 불일치 → AUTH_001. 탈퇴 시 사진·도면·공간·추천안·공유링크·분석잡·사용자까지 전부 삭제.
- **결정 3 (keepResults)**: 원본 삭제 시 `keepResults=true`면 추천 텍스트·AI 생성 이미지 유지(원본만 삭제, 전후비교는 After 단독). `false`면 연결된 추천안·공유링크까지 삭제.
- **결정 4 (진행 중 분석 보호)**: 삭제 대상 공간에 진행 중(QUEUED~GENERATING_VISUAL) 분석이 있으면 AI_005로 차단.
- ID는 Long.

## 엔드포인트

### GET /me/profile

- **기능**: FR MY-001 마이페이지 (프로필 + 요약 통계)
- **응답 200**:
  ```json
  { "success": true, "data": {
    "userId": 1, "email": "user@example.com", "nickname": "홍길동",
    "consents": { "termsOfService": true, "privacyPolicy": true, "imageProcessing": true, "marketing": false },
    "stats": { "spaceCount": 2, "savedRecommendationCount": 3, "photoCount": 7 } } }
  ```
- **에러**: 없음(인증 실패는 공통 401)

### PATCH /me

- **기능**: MY-003 계정 설정 — 닉네임·마케팅 동의 변경
- **요청**: `{ "nickname": "새닉네임", "marketing": true }` (둘 다 nullable — 전달된 필드만 수정)
  - nickname: 1~20자
- **응답 200**: `{ "success": true, "data": { "userId": 1, "nickname": "새닉네임", "marketing": true } }`
- **에러**: VALID_001(닉네임 형식)

### GET /me/images

- **기능**: FR-DATA-001 업로드 원본 전체 조회 (MY-002). 모든 공간의 사진·도면 통합 목록
- **응답 200**:
  ```json
  { "success": true, "data": { "items": [
    { "photoId": 10, "spaceId": 1, "spaceName": "거실", "url": "/files/ab12.jpg",
      "isFloorPlan": false, "uploadedAt": "2026-07-20T10:00:00Z" }
  ] } }
  ```
  - 빈 목록 `items: []`
- **에러**: 없음

### DELETE /me/images

- **기능**: FR-DATA-002/003 원본 삭제(개별/일괄). 스토리지 파일 물리 삭제
- **요청**:
  ```json
  { "imageIds": [10, 11], "deleteAll": false, "keepResults": true, "confirmShareRevoke": false }
  ```
  - `imageIds`: 개별 삭제 대상(deleteAll=false일 때 필수). `deleteAll=true`면 전체 원본
  - `keepResults` 필수: true면 추천 결과 유지·원본만 삭제, false면 연결 추천안·공유링크까지 삭제
  - `confirmShareRevoke`: 원본 포함 공유링크가 있을 때 회수 동의. false인데 해당 링크 존재 시 SHARE_002로 선차단(사용자 확인 후 true로 재요청)
- **응답 200**:
  ```json
  { "success": true, "data": { "deletedCount": 2, "revokedShareLinks": 1, "deletedRecommendations": 0 } }
  ```
- **에러**: SHARE_002(원본 포함 공유링크 존재 — confirmShareRevoke=false), AI_005(대상 공간에 진행 중 분석), AUTH_003(타인 이미지 포함), RES_001(없는 imageId), VALID_001(imageIds·deleteAll 둘 다 없음)

### DELETE /me  (회원 탈퇴)

- **기능**: FR-DATA-004 탈퇴 — 모든 개인 데이터 + 계정 삭제
- **요청**: `{ "password": "Passw0rd!" }`
- **처리**: 사진·도면 스토리지 파일 물리 삭제 → 공간·추천안·공유링크·분석잡·refreshToken·사용자 삭제. 되돌릴 수 없음
- **응답 200**: `{ "success": true, "data": { "deleted": true } }`
- **에러**: AUTH_001(비밀번호 불일치), AI_005(진행 중 분석 존재 — 완료/취소 후 탈퇴)

## 에러 코드

| 코드 | 의미 |
|------|------|
| AUTH_001 | 인증 실패(탈퇴 시 비밀번호 불일치) |
| AUTH_003 | 접근 권한 없음(타인 이미지) |
| RES_001 | 리소스 없음 |
| VALID_001 | 입력값 누락/형식(닉네임, 삭제 대상 미지정) |
| SHARE_002 | 원본 포함 공유링크 존재(확인 후 회수 진행) |
| AI_005 | 분석 진행 중 삭제/탈퇴 충돌 |

## 삭제 연쇄 규칙 (구현 주의)

- `DELETE /me/images` (keepResults=true): SpacePhoto 행 + 스토리지 파일만 삭제. Recommendation/ShareLink 유지. 단 원본 포함 공유링크는 confirmShareRevoke=true 시 revoked 처리.
- `DELETE /me/images` (keepResults=false): 위 + 해당 사진이 속한 공간의 Recommendation·연결 ShareLink 삭제.
- `DELETE /me` (탈퇴): 사용자의 모든 SpacePhoto 파일 물리 삭제 후 Space/Recommendation/ShareLink/AnalysisJob/RefreshToken/User 전부 삭제. 트랜잭션으로 원자적 처리, 파일 삭제 실패해도 DB 정리는 진행(고아 파일 로그).
- **물리 삭제 검증 필수**: QA가 실제 업로드 파일이 스토리지에서 사라지는지 확인해야 함(NFR-SEC-004).
