# M1 계약 — AUTH + HOME

공통 규칙은 `api-contract-guide.md` 표준을 따른다 (Base URL `/api/v1`, ApiResponse 포맷, camelCase, `Authorization: Bearer {accessToken}`).

## 범위

- FR-AUTH-002 이메일 회원가입/로그인, FR-AUTH-003 이미지 데이터 처리 동의, FR-AUTH-005 자동 로그인(토큰 갱신)
- FR-AUTH-001 소셜 로그인: 카카오·Google 실연동(2026-07-20). 네이버·Apple 은 범위 제외
- 화면: COM-001 스플래시, COM-002 온보딩, COM-003 로그인, COM-004 약관 동의, COM-005 권한 안내, HOME-001 홈, HOME-002 스타일 갤러리, HOME-003 스타일 상세

## AUTH 엔드포인트

### POST /auth/signup

- **기능**: FR-AUTH-002 이메일 회원가입 (약관 동의 포함 원스텝 가입)
- **인증**: 불필요
- **요청**:
  ```json
  { "email": "user@example.com", "password": "Passw0rd!", "nickname": "홍길동",
    "consents": { "termsOfService": true, "privacyPolicy": true, "imageProcessing": true, "aiTraining": false, "marketing": false } }
  ```
  - `password`: 8~20자, 영문+숫자 포함
  - `consents.termsOfService`/`privacyPolicy`/`imageProcessing`: **필수** true (하나라도 false면 AUTH_004)
  - `consents.aiTraining`/`marketing`: **선택**(false 허용). AI 학습 활용은 서비스 제공에 불필요하므로 필수로 강제하지 않음 — 개인정보보호법 제22조(끼워팔기 동의 금지) 및 NFR-SEC-005 준수. 상세: `docs/privacy_consent.md`
  - 가입 화면은 만 14세 이상 확인(필수, 클라이언트 게이트)을 포함
- **응답 200** (refreshToken 은 본문이 아닌 httpOnly 쿠키 — 토큰 정책 절 참조):
  ```json
  { "success": true, "data": { "accessToken": "eyJ...", "user": { "userId": 1, "email": "user@example.com", "nickname": "홍길동" } } }
  ```
- **에러**: AUTH_004(필수 동의 미완료), VALID_001(이메일 형식/중복, 비밀번호 형식)

### POST /auth/login

- **기능**: FR-AUTH-002 이메일 로그인
- **인증**: 불필요
- **요청**: `{ "email": "user@example.com", "password": "Passw0rd!" }`
- **응답 200** (refreshToken 은 httpOnly 쿠키):
  ```json
  { "success": true, "data": { "accessToken": "eyJ...", "user": { "userId": 1, "email": "user@example.com", "nickname": "홍길동" } } }
  ```
- **에러**: AUTH_001(이메일 또는 비밀번호 불일치), AUTH_005(로그인 실패 15분 내 5회 초과 — 429)

### POST /auth/kakao · POST /auth/google

- **기능**: FR-AUTH-001 소셜 로그인 — FE 가 제공자 인가 URL로 리다이렉트시킨 뒤, 콜백에서 받은 인가 코드를 BE 로 전달하면 BE 가 토큰 교환·사용자 조회·자동가입까지 처리
- **인증**: 불필요
- **요청**: `{ "code": "인가 코드" }`
- **응답 200** (signup/login과 동일 포맷, refreshToken 은 httpOnly 쿠키):
  ```json
  { "success": true, "data": { "accessToken": "eyJ...", "user": { "userId": 1, "email": "...", "nickname": "..." } } }
  ```
- **동작**:
  - 최초 로그인 시 자동 가입. 필수 동의 3종은 소셜 제공자의 로그인 동의 화면이 대신한다고 간주해 true로 저장(별도 동의 화면 없음 — v1 간소화, 백로그)
  - 카카오: 비즈 앱 미전환 상태라 이메일 동의항목 사용 불가 → `providerId` 기반 더미 이메일(`kakao_{id}@social.homestyler.local`) 발급. 닉네임만 실제 값
  - Google: 이메일·이름 모두 실제 값 사용
  - **계정 연동**: 검증된 이메일이 기존 유저(로컬 또는 타 제공자)와 일치하면 새 계정을 만들지 않고 그 유저에 이 제공자를 연결한다(`social_accounts` 행 추가). 이후 로컬 비밀번호·연동된 모든 소셜 제공자로 동일 계정에 로그인 가능. 카카오 더미 이메일은 provider+providerId 조합이라 실제 유저와 절대 충돌하지 않으므로 연동 대상에서 자연히 제외
- **에러**: AUTH_006(토큰 교환/사용자 조회 실패), VALID_001(이메일 충돌)
- **FE 콜백**: `/oauth/callback/{provider}` 라우트가 `?code=`를 받아 이 엔드포인트를 호출. 인가 URL 조립은 `frontend/src/api/oauth.ts` (client_id/REST API 키는 공개 값이라 FE에 둠 — client_secret 은 BE 전용)

### POST /auth/refresh

- **기능**: FR-AUTH-005 자동 로그인 — accessToken 만료 시 refreshToken으로 재발급
- **인증**: 불필요 (refreshToken 은 httpOnly 쿠키로 자동 전송 — 본문 없음)
- **요청**: `{}` (빈 본문. 쿠키 `refreshToken` 필수)
- **응답 200**: `{ "success": true, "data": { "accessToken": "eyJ..." } }` + 회전된 refreshToken 쿠키 재발급
- **에러**: AUTH_002(refreshToken 만료/무효/쿠키 없음 — FE는 로그인 화면으로 이동)

### POST /auth/logout

- **기능**: 로그아웃 — 서버 refreshToken 폐기 + httpOnly 쿠키 삭제
- **인증**: 필요
- **응답 200**: `{ "success": true }`

### GET /auth/me

- **기능**: 자동 로그인 시 저장된 accessToken으로 현재 사용자 조회(스플래시에서 세션 확인용)
- **인증**: 필요
- **응답 200**: `{ "success": true, "data": { "userId": 1, "email": "user@example.com", "nickname": "홍길동" } }`
- **에러**: AUTH_002(토큰 만료), AUTH_001(토큰 무효)

### POST /auth/consents

- **기능**: FR-AUTH-003 동의 항목 갱신(마케팅 동의 등 가입 후 변경용). 가입 시점 동의는 `/auth/signup`에 포함되므로 이 엔드포인트는 가입 후 변경 전용
- **인증**: 필요
- **요청**: `{ "marketing": true }`
- **응답 200**: `{ "success": true, "data": { "marketing": true } }`
- **에러**: 없음

## HOME 엔드포인트

### GET /home/summary

- **기능**: HOME-001 홈 진입 데이터 (인사말 + 최근 공간 + 추천 배너 + 스타일 프리뷰)
- **인증**: 필요 (FR-AUTH-004 비회원 둘러보기는 P2 — v1은 로그인 필요, 미인증 시 401)
- **응답 200**:
  ```json
  { "success": true, "data": {
    "nickname": "홍길동",
    "recentSpaces": [ { "spaceId": 1, "name": "거실", "thumbnailUrl": "/files/sp1.jpg" } ],
    "styleHighlights": [ { "styleType": "NORDIC", "title": "북유럽 스타일", "thumbnailUrl": "/files/style_nordic.jpg" } ]
  } }
  ```
- **에러**: 없음(공간 없으면 `recentSpaces: []`)

### GET /styles

- **기능**: HOME-002 스타일 갤러리 목록
- **인증**: 불필요
- **응답 200**:
  ```json
  { "success": true, "data": { "items": [
    { "styleType": "NORDIC", "title": "북유럽 스타일", "thumbnailUrl": "/files/style_nordic.jpg", "description": "밝은 우드톤과 화이트 배색" }
  ] } }
  ```
- **에러**: 없음

### GET /styles/{styleType}

- **기능**: HOME-003 스타일 상세
- **인증**: 불필요
- **응답 200**:
  ```json
  { "success": true, "data": {
    "styleType": "NORDIC", "title": "북유럽 스타일", "description": "...",
    "keywords": ["화이트", "우드", "미니멀"],
    "gallery": [ { "imageUrl": "/files/style_nordic_1.jpg", "caption": "거실 예시" } ]
  } }
  ```
- **에러**: RES_001(존재하지 않는 styleType)

## Enums

- StyleType: MODERN | MINIMAL | NATURAL | NORDIC | HOTEL | WOOD

## 에러 코드

| 코드 | 의미 |
|------|------|
| AUTH_001 | 인증 실패(이메일/비밀번호 불일치, 토큰 무효) |
| AUTH_002 | 토큰 만료 (accessToken 401 → FE가 /auth/refresh 1회 재시도, 실패 시 로그인 화면 이동) |
| AUTH_004 | 필수 동의 미완료 |
| AUTH_005 | 로그인 시도 초과(이메일당 15분 창 5회 실패 → 429) |
| AUTH_006 | 소셜 로그인 실패(제공자 토큰 교환/사용자 조회 오류) |
| VALID_001 | 입력값 의미 오류(이메일 중복 등 — DB 조회 필요한 검증) |
| COMMON_400 | 입력값 형식 오류(비밀번호 형식 등 — `@Valid` bean validation 실패, GlobalExceptionHandler 공통 처리) |
| RES_001 | 리소스 없음 |

## 토큰 정책

- accessToken 만료: 1시간, refreshToken 만료: 30일 (JWT, HS256)
- refreshToken 은 **httpOnly 쿠키**(`SameSite=Strict`, `Path=/api/v1/auth`, prod 는 `Secure`)로만 오간다 — 응답 본문·localStorage 에 두지 않는다(XSS 탈취 방지, 2026-07-20 보안 점검 High-5). signup/login/refresh 가 발급·회전하고 logout 이 삭제한다.
- FE: accessToken 만 localStorage 저장, 앱 시작 시(COM-001 스플래시) `/auth/me` 호출로 세션 확인 → 실패 시 refresh 시도 → 둘 다 실패 시 온보딩/로그인으로 이동
- 업로드 파일 서빙(`/files/{filename}`)은 HMAC 서명 쿼리(`exp`,`sig`) 필수 — API 응답의 URL 을 그대로 사용(24시간 유효). 무서명/변조/만료는 403 (보안 점검 High-2)
