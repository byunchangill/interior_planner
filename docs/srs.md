## 1. 문서 개요

### 1.1 목적

본 문서는 AI 기반 맞춤형 인테리어 추천 모바일 앱 "HomeStyler(홈스타일러)"가 충족해야 할 기능 및 비기능 요구사항을 정의한다. 사용자가 자신의 집 사진 또는 도면을 업로드하고 원하는 스타일·예산·조건을 선택하면, 공간의 구조·크기·채광·동선·기존 가구를 분석해 맞춤형 인테리어를 추천하는 서비스의 개발 범위를 명확히 하는 것이 목적이다.

기획자·개발자·디자이너·QA가 공통의 기준으로 삼을 수 있도록, "무엇을" 만들어야 하는지를 기술한다. "어떻게" 동작하는지는 기능 명세서(functional-spec.md)에서 다룬다.

### 1.2 범위

- **서비스명**: HomeStyler (약칭: HS)
- **포함 범위**: 본 문서는 HomeStyler 모바일 앱(iOS/Android)과 이를 지원하는 백엔드 API, 관리자 웹 백오피스의 요구사항을 대상으로 한다.
- **제외 범위**: 실제 시공 중개·결제, 가구 판매(커머스) 기능은 v1.0 범위에서 제외한다. 가구 추천은 정보 제공 및 외부 링크 수준으로 한정한다.

### 1.3 용어 정의

| 용어 | 정의 |
|------|------|
| 공간(Space) | 사용자가 등록한 개별 방 단위(거실, 침실, 주방, 서재 등) |
| 공간 분석 | 업로드된 사진/도면에서 구조, 크기, 채광, 동선, 가구를 AI가 추출하는 과정 |
| 추정 치수 | AI가 사진/도면에서 산출한 공간 치수. 실측치와 다를 수 있으며 사용자가 수정 가능 |
| 스타일 | 인테리어 콘셉트 분류(모던, 미니멀, 내추럴, 북유럽, 호텔식, 우드톤) |
| 추천안(Recommendation) | 하나의 공간 × 하나의 스타일 조합으로 생성된 인테리어 추천 결과 세트 |
| 전후 비교 이미지 | 현재 공간 사진과 AI가 생성한 인테리어 적용 시각화 이미지의 비교 뷰 |
| 유지 가구 | 사용자가 새 인테리어에서도 계속 사용하겠다고 지정한 기존 가구 |
| 전문가 확인 대상 제안 | 구조 변경, 전기, 배관, 철거 등 전문 시공 검토가 필요한 추천 항목 |
| 원본 데이터 | 사용자가 업로드한 사진·도면 파일의 원본 |
| 게스트(G) | 비회원 상태로 서비스를 탐색하는 사용자 |

## 2. 시스템 개요

### 2.1 제품 관점

HomeStyler는 독립형 모바일 서비스로, 다음 외부 시스템과 연계된다.

```
[모바일 앱 (iOS/Android)]
        │
        ▼
[HomeStyler API 서버] ──── [AI 분석 엔진] (공간 구조/채광/동선/가구 인식)
        │                  [AI 이미지 생성] (인테리어 시각화)
        │
        ├── [소셜 로그인] 카카오 / 네이버 / Google
        ├── [푸시 알림] FCM / APNs
        ├── [오브젝트 스토리지] 사진·도면 암호화 저장
        └── [공유 링크 서비스] 보기 전용 웹 뷰
```

### 2.2 사용자 클래스 및 특성

| 사용자 유형 | 설명 | 기술 수준 | 주요 목적 |
|------------|------|-----------|-----------|
| 일반 사용자 | 거주 중 또는 이사 예정인 성인. 인테리어 전문 지식 없음 | 낮음~중간 | 내 공간에 맞는 인테리어를 쉽고 빠르게 결정 |
| 공유 수신자 | 가족, 인테리어 업체 등 링크로 결과를 열람하는 사람 | 낮음 | 추천안 열람 및 의견 교환 |
| 관리자 | 서비스 운영자 | 높음 | 콘텐츠·가구 DB 관리, 문의 처리, 지표 모니터링 |

### 2.3 운영 환경

- 플랫폼: iOS 16+, Android 12+
- 관리자 백오피스: 웹 (Chrome 최신)
- 네트워크: WiFi 및 모바일 데이터 지원. 분석 요청은 온라인 필수, 저장된 결과는 오프라인 열람 가능(캐시)
- 서버: 클라우드 환경 (AI 분석·이미지 생성은 GPU 워커 분리 운영)

### 2.4 제약사항

- **법적/규제**: 개인정보보호법 준수. 집 내부 사진·도면은 개인의 생활공간 정보로서 민감하게 취급하며, 수집·이용·삭제 정책을 명시적으로 고지·동의받아야 한다.
- **AI 한계 고지 의무**: 추천 결과가 실제 치수·시공 가능 여부와 다를 수 있음을 결과 화면마다 고지해야 한다.
- **기술적**: AI 이미지 생성은 비동기 처리(수십 초~수 분 소요 가능). 동기 응답 UX로 설계 불가.
- **사업적**: v1.0은 추천·시각화·공유까지. 시공 중개/커머스는 후속 버전.

## 6. 제약사항 및 가정

### 제약사항
- AI 추정 치수·시각화 결과는 참고용이며 실측·시공 판단의 근거가 될 수 없다. 이 한계는 UI 전반에 고지한다.
- 구조 변경·전기·배관·철거 관련 제안은 정보 제공에 한하며, 시공 가능 여부 판단은 전문가 몫임을 명시한다.
- v1.0에서는 결제·시공 중개 기능을 포함하지 않는다.

### 가정
- 사용자는 스마트폰 카메라로 공간을 촬영할 수 있는 환경에 있다.
- AI 분석 엔진과 이미지 생성 모델은 별도 조달 또는 외부 API 활용이 가능하다.
- 가구 추천 DB는 초기에는 관리자 수동 구축, 이후 제휴 데이터 연동을 검토한다.
