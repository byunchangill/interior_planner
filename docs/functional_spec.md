## FR-AUTH-001 소셜 로그인

**관련 화면**: COM-003
**우선순위**: [P1]
**관련 요구사항**: FR-AUTH-001, FR-AUTH-003, FR-AUTH-005

### 개요
> 카카오/네이버/Apple/Google 계정으로 로그인하고, 최초 가입 시 이미지 데이터 처리 동의를 받는다.

### 트리거
- 사용자가 로그인 화면에서 소셜 로그인 버튼을 탭했을 때

### 입력값
| 파라미터 | 타입 | 필수 | 유효성 검사 |
|----------|------|------|-------------|
| provider | string | Y | kakao / naver / apple / google |
| authCode | string | Y | 소셜 플랫폼 발급 인가 코드 |
| deviceId | string | Y | 기기 고유 식별자 |

### 처리 로직
1. 클라이언트가 소셜 SDK로 인가 코드 획득
2. 서버가 소셜 플랫폼에 토큰 교환 요청 → 사용자 식별자·이메일 획득
3. 기존 회원이면 로그인 처리, 신규면 회원 생성 후 `needsConsent: true` 반환
4. 신규 회원은 약관·이미지 데이터 처리 동의 화면으로 이동, 동의 완료 시 가입 확정
5. Access Token(1시간) + Refresh Token(30일) 발급

### 출력/결과
- 성공 시: 토큰 발급 후 홈 이동 (신규는 권한 요청 안내 → 홈)
- 실패 시: 에러 메시지 토스트, 로그인 화면 유지

### 예외 처리
| 예외 상황 | 에러 코드 | 사용자 메시지 |
|-----------|-----------|---------------|
| 소셜 인증 실패/취소 | AUTH_001 | 로그인에 실패했습니다. 다시 시도해 주세요. |
| 필수 동의 미완료 | AUTH_004 | 필수 약관에 동의해야 서비스를 이용할 수 있습니다. |
| 토큰 만료 | AUTH_002 | (자동 갱신, 실패 시) 다시 로그인해 주세요. |

### API 엔드포인트 (안)
```
POST /api/v1/auth/social-login
Request: { "provider": "kakao", "authCode": "...", "deviceId": "..." }
Response: { "success": true, "data": { "accessToken": "...", "refreshToken": "...", "isNewUser": true, "needsConsent": true } }

POST /api/v1/auth/consents
Request: { "termsOfService": true, "privacyPolicy": true, "imageProcessing": true, "marketing": false }
```

### Use Case 시나리오
**정상 시나리오:**
1. 사용자가 카카오 로그인 버튼 탭 → 카카오 인증 완료
2. 신규 회원으로 판별 → 약관·이미지 처리 동의 화면 표시
3. 필수 동의 완료 → 가입 확정 → 카메라/사진첩 권한 안내 → 홈 진입

**예외 시나리오:**
1. 사용자가 이미지 데이터 처리 동의(필수)를 거부 → 서비스 핵심 기능 이용 불가 안내와 함께 가입 중단

## FR-SPACE-002 사진 촬영 등록

**관련 화면**: SPACE-002, SPACE-003
**우선순위**: [P1]
**관련 요구사항**: FR-SPACE-001, FR-SPACE-002, FR-SPACE-003, FR-SPACE-008

### 개요
> 촬영 가이드와 함께 공간 사진을 촬영·업로드하고, 품질 검증을 통과한 사진만 등록한다.

### 트리거
- 공간 유형 선택 후 "사진 촬영" 또는 "사진첩에서 선택"을 탭했을 때

### 입력값
| 파라미터 | 타입 | 필수 | 유효성 검사 |
|----------|------|------|-------------|
| spaceType | string | Y | living / bedroom / kitchen / study / etc |
| images[] | file | Y | JPG/PNG/HEIC, 장당 최대 20MB, 공간당 1~10장 |
| homeId | string | Y | 사용자 소유 집 ID |

### 처리 로직
1. 촬영 화면에 가이드 오버레이 표시(공간 전체가 보이는 코너 각도, 1.5m 이상 거리, 조명 켜기 팁)
2. 촬영/선택된 이미지의 EXIF GPS 정보를 클라이언트에서 제거
3. 클라이언트 리사이즈(장변 2560px) 후 서명 URL로 스토리지 직접 업로드
4. 서버가 품질 검증 실행: 블러 점수, 밝기, 공간 인식 여부
5. 품질 미달 이미지는 사유와 함께 재촬영 안내, 통과 이미지는 공간에 연결
6. 등록 완료 시 치수 추정 단계(SPACE-005)로 이동

### 출력/결과
- 성공 시: 업로드된 사진 썸네일 목록 표시 → "다음: 치수 확인" 버튼 활성화
- 실패 시: 품질 미달 사유 표시(예: "사진이 너무 어두워요. 조명을 켜고 다시 촬영해 주세요")

### 예외 처리
| 예외 상황 | 에러 코드 | 사용자 메시지 |
|-----------|-----------|---------------|
| 카메라 권한 거부 | PERM_001 | 카메라 권한이 필요합니다. 설정에서 허용해 주세요. |
| 파일 형식/용량 초과 | VALID_002 | 지원하지 않는 형식이거나 용량이 20MB를 초과합니다. |
| 품질 검증 실패 | IMG_001 | 공간을 인식하기 어려워요. 가이드에 맞춰 다시 촬영해 주세요. |
| 업로드 네트워크 오류 | NET_001 | 업로드에 실패했습니다. 네트워크를 확인해 주세요. (재시도 버튼) |
| 장수 초과 | VALID_003 | 공간당 최대 10장까지 등록할 수 있습니다. |

### API 엔드포인트 (안)
```
POST /api/v1/spaces
Request: { "homeId": "...", "spaceType": "living" }
Response: { "success": true, "data": { "spaceId": "sp_123" } }

POST /api/v1/spaces/{spaceId}/images/upload-url
Request: { "fileName": "living_01.jpg", "contentType": "image/jpeg" }
Response: { "success": true, "data": { "uploadUrl": "https://...", "imageId": "img_456" } }

POST /api/v1/spaces/{spaceId}/images/{imageId}/confirm
Response: { "success": true, "data": { "qualityCheck": "passed" } }
```

### Use Case 시나리오
**정상 시나리오:**
1. 사용자가 "거실" 선택 → "사진 촬영" 탭
2. 가이드에 맞춰 코너 각도로 3장 촬영
3. 품질 검증 통과 → 썸네일 확인 → 치수 확인 단계로 이동

**예외 시나리오:**
1. 저조도 사진 업로드 → "조명을 켜고 다시 촬영해 주세요" 안내 → 재촬영 후 통과

## FR-SPACE-005 치수 추정 및 수정

**관련 화면**: SPACE-005
**우선순위**: [P1]
**관련 요구사항**: FR-SPACE-005, FR-RECO-012

### 개요
> AI가 추정한 공간 치수를 사용자가 확인·수정하는 신뢰 확보 단계. 추정치임을 명확히 고지한다.

### 트리거
- 사진/도면 등록 완료 후 자동 진입, 또는 공간 상세에서 "치수 수정" 탭

### 입력값
| 파라미터 | 타입 | 필수 | 유효성 검사 |
|----------|------|------|-------------|
| width | number | Y | 1.0~20.0 (m), 소수 첫째 자리 |
| depth | number | Y | 1.0~20.0 (m) |
| height | number | N | 2.0~5.0 (m), 미입력 시 2.3m 기본값 |
| openings[] | array | N | 창문/문 위치·크기 |
| unit | string | N | m / cm / 평 (표시 단위) |

### 처리 로직
1. AI 추정 결과를 신뢰도(높음/중간/낮음)와 함께 표시
2. 화면 상단에 고정 배너: "AI 추정치입니다. 실제 치수와 다를 수 있으니 확인 후 수정해 주세요"
3. 사용자가 값 수정 시 `isUserVerified: true` 플래그 저장 (추천 정확도 가중치에 활용)
4. 도면 업로드 케이스는 도면 축척 기준 자동 산출값 우선 표시
5. 창문·문 위치는 평면 스케치 위에서 드래그로 조정

### 출력/결과
- 성공 시: 치수 확정 → 기존 가구 등록 단계(SPACE-006)로 이동
- 실패 시: 유효 범위 밖 값 입력 시 인라인 에러 표시

### 예외 처리
| 예외 상황 | 에러 코드 | 사용자 메시지 |
|-----------|-----------|---------------|
| 치수 범위 초과 | VALID_003 | 1m~20m 범위로 입력해 주세요. |
| AI 추정 실패 | AI_002 | 치수를 추정하지 못했어요. 직접 입력해 주세요. (수동 입력 폼 활성화) |

### API 엔드포인트 (안)
```
PATCH /api/v1/spaces/{spaceId}/dimensions
Request: { "width": 4.5, "depth": 3.6, "height": 2.3, "unit": "m", "isUserVerified": true,
           "openings": [{ "type": "window", "wall": "north", "width": 1.8 }] }
Response: { "success": true, "data": { "spaceId": "sp_123", "areaPyeong": 4.9 } }
```

### Use Case 시나리오
**정상 시나리오:**
1. AI가 거실을 4.2m × 3.5m로 추정(신뢰도: 중간) 표시
2. 사용자가 실측 기준 4.5m × 3.6m로 수정 → 확정
3. 기존 가구 선택 단계로 이동

**예외 시나리오:**
1. 도면 없이 근접 사진만 업로드해 추정 실패 → 수동 입력 폼 표시 → 사용자가 직접 입력

## FR-RECO-001 공간 분석 실행

**관련 화면**: RECO-001, RECO-002, RECO-003
**우선순위**: [P1]
**관련 요구사항**: FR-PREF-001~004, FR-RECO-001, FR-RECO-013

### 개요
> 공간 데이터 + 사용자 조건(스타일/예산/색상/가구)으로 AI 분석을 비동기 실행한다.

### 트리거
- 조건 설정 완료 후 "맞춤 인테리어 분석 시작" 버튼을 탭했을 때

### 입력값
| 파라미터 | 타입 | 필수 | 유효성 검사 |
|----------|------|------|-------------|
| spaceId | string | Y | 본인 소유 공간, 사진·치수 등록 완료 상태 |
| styles[] | array | Y | 1~3개 (modern/minimal/natural/nordic/hotel/wood) |
| budgetRange | string | Y | 정의된 예산 구간 코드 |
| preferredColors[] | array | N | 색상 코드 목록 |
| requiredFurniture[] | array | N | 가구 카테고리 코드 |
| keepFurnitureIds[] | array | N | 유지 가구 ID (SPACE-006에서 선택된 값) |
| lifestyle | object | Y | 생활 방식 설문 (FR-PREF-006): householdSize(1~10), hasChildren, hasPets, worksFromHome, cooksOften, storagePreference(STORAGE/OPENNESS), housingType(JEONSE/WOLSE/OWNED), residenceYears |

### 처리 로직
1. 입력 검증 후 분석 Job 생성 → 분석 큐에 등록 (상태: `queued`)
2. GPU 워커가 단계별 분석 수행, 단계마다 진행 상태 갱신:
   - `analyzing_structure` (구조·치수 분석)
   - `analyzing_light` (채광 분석: 창 방향·크기 기반)
   - `analyzing_flow` (동선 분석)
   - `generating_reco` (스타일별 추천 생성)
   - `generating_visual` (전후 비교 시각화 생성)
3. 클라이언트는 폴링(5초) 또는 푸시로 상태 수신, 진행률 UI 표시
4. 완료 시 스타일별 추천안 생성(선택 스타일 수만큼), 푸시 알림 발송
5. 실패 시 자동 재시도 최대 2회, 최종 실패 시 실패 알림 + 재요청 버튼

### 출력/결과
- 성공 시: `completed` 상태 + 추천안 ID 목록 → 결과 요약 화면 자동 이동(또는 알림 탭 진입)
- 실패 시: 실패 사유 안내, 조건 수정 후 재시도 유도

### 예외 처리
| 예외 상황 | 에러 코드 | 사용자 메시지 |
|-----------|-----------|---------------|
| 공간 데이터 불완전 | VALID_001 | 공간 사진과 치수를 먼저 등록해 주세요. |
| 스타일 미선택/초과 | VALID_003 | 스타일은 1~3개 선택할 수 있습니다. |
| 분석 워커 최종 실패 | AI_001 | 분석에 실패했어요. 잠시 후 다시 시도해 주세요. |
| 분석 대기 초과 | AI_003 | 요청이 많아 지연되고 있어요. 완료되면 알려드릴게요. |
| 동시 분석 제한 | LIMIT_001 | 진행 중인 분석이 완료된 후 새 분석을 시작할 수 있어요. |

### API 엔드포인트 (안)
```
POST /api/v1/analyses
Request: { "spaceId": "sp_123", "styles": ["nordic", "wood"], "budgetRange": "B300_500",
           "preferredColors": ["#F5F0E8", "#8B7355"], "requiredFurniture": ["sofa", "bookshelf"],
           "keepFurnitureIds": ["fur_001"],
           "lifestyle": { "householdSize": 2, "hasChildren": false, "hasPets": true,
                          "worksFromHome": true, "cooksOften": false,
                          "storagePreference": "STORAGE", "housingType": "JEONSE", "residenceYears": 2 } }
Response: { "success": true, "data": { "analysisId": "an_789", "status": "queued", "estimatedSeconds": 120 } }

GET /api/v1/analyses/{analysisId}
Response: { "success": true, "data": { "status": "generating_visual", "progress": 80,
            "recommendationIds": [] } }
```

### Use Case 시나리오
**정상 시나리오:**
1. 사용자가 북유럽+우드톤 스타일, 예산 300~500만 원 선택 → 분석 시작
2. 진행률 화면에서 "채광 분석 중..." 등 단계 표시 확인
3. 2분 후 완료 푸시 수신 → 결과 요약 화면 진입

**예외 시나리오:**
1. 분석 중 앱 종료 → 완료 시 푸시 수신 → 알림 탭으로 결과 진입

## FR-RECO-002~009 통합 추천 결과 제공

**관련 화면**: RECO-004, RECO-005, RECO-007
**우선순위**: [P1]
**관련 요구사항**: FR-RECO-002~009, FR-RECO-012

### 개요
> 분석 완료된 추천안의 8개 섹션(콘셉트/배치·동선/마감재/확장 팁/수납/기존 가구 배치/예산별/추천 가구)을 제공한다.

### 트리거
- 결과 요약에서 스타일 탭 선택 후 "상세 보기" 탭

### 입력값
| 파라미터 | 타입 | 필수 | 유효성 검사 |
|----------|------|------|-------------|
| recommendationId | string | Y | 본인 소유 추천안 |
| section | string | N | 특정 섹션 앵커 이동용 |

### 처리 로직
1. 추천안 상세 데이터를 섹션 구조로 조회 (1회 호출, 클라이언트 캐시)
2. 화면 최상단에 AI 한계 고지 배너 고정: "본 추천은 AI 분석 결과로 실제 치수·시공 가능 여부와 다를 수 있습니다"
3. 섹션 구성:
   - `concept`: 콘셉트명, 설명, 무드 키워드
   - `layout`: 배치도 이미지(top-view), 동선 설명, 배치 근거
   - `materials`: 벽지/바닥/조명/커튼 각각의 색상(HEX)·소재·추천 이유
   - `spaceTips`: 공간 확장 팁 목록
   - `storage`: 수납 활용 제안 목록
   - `keepFurnitureLayout`: 유지 가구 반영 배치안 (유지 가구 없으면 섹션 숨김)
   - `budgetPlans`: 예산 구간별 추천 구성 (설정 예산 구간 기본 펼침)
   - `items`: 추천 가구·소품 목록 — 실제 판매 제품 형태의 스키마(브랜드, 제품명, W×D×H 치수, 가격, 구매 링크, 배치 위치 태그). v1은 Mock 값이지만 스키마는 실데이터 교체 가능 형태(BL-001)로 유지
   - `fitScore`: 공간 적합도(FR-RECO-015) — 총점(0~100) + 항목별 판정(GOOD/CHECK/BLOCKED, 예: 통로 폭, 가구 대비 벽 길이) + 구매 전 실측 확인 목록
4. 구조 변경·전기·배관·철거 관련 항목에는 `expertRequired: true` 플래그 → 경고 배지 표시, 탭 시 RECO-008 안내로 이동
5. 배치·마감재·가구 등 주요 추천 항목에는 `reason`(추천 이유, FR-RECO-014)을 포함해 표시한다. 생활 방식 설문(lifestyle)이 반영된 항목은 이유에 그 근거를 명시한다 (예: "반려동물 가정 — 긁힘에 강한 패브릭 추천")

### 출력/결과
- 성공 시: 섹션 탐색형 상세 화면 렌더링, 항목별 스크랩 가능
- 실패 시: 결과 로드 실패 안내 + 재시도

## FR-DATA-002 원본 데이터 삭제

**관련 화면**: MY-002
**우선순위**: [P1]
**관련 요구사항**: FR-DATA-001~004, NFR-SEC-004

### 개요
> 업로드한 사진·도면 원본을 개별/일괄로 영구 삭제한다. 개인 공간 데이터에 대한 사용자 통제권의 핵심.

### 트리거
- 데이터·개인정보 관리 화면에서 삭제 버튼 탭

### 입력값
| 파라미터 | 타입 | 필수 | 유효성 검사 |
|----------|------|------|-------------|
| imageIds[] | array | Y(개별) | 본인 소유 이미지 |
| deleteAll | boolean | N | true 시 전체 삭제 |
| keepResults | boolean | Y | 추천 결과 유지 여부 |

### 처리 로직
1. 삭제 확인 다이얼로그: 삭제 대상 개수, "삭제 후 복구 불가", 추천 결과 유지 여부 선택 제시
2. `keepResults: true`면 추천 텍스트·AI 생성 이미지는 유지, 원본만 삭제 (전후 비교는 After 단독 뷰로 전환)
3. `keepResults: false`면 연결된 추천 결과·공유 링크까지 함께 삭제
4. 스토리지 원본 즉시 삭제 + 백업본은 30일 내 파기 스케줄 등록
5. 삭제 완료 이력(무엇을·언제 삭제했는지) 기록, 사용자에게 완료 안내

### 출력/결과
- 성공 시: "N장의 원본이 삭제되었습니다" 완료 안내, 목록 갱신
- 실패 시: 부분 실패 시 실패 항목 표시 + 재시도

### 예외 처리
| 예외 상황 | 에러 코드 | 사용자 메시지 |
|-----------|-----------|---------------|
| 공유 링크에 원본 포함됨 | SHARE_002 | 원본이 포함된 공유 링크가 있어요. 삭제 시 해당 링크도 회수됩니다. (확인 후 진행) |
| 분석 진행 중 삭제 시도 | AI_005 | 분석이 진행 중이에요. 완료 또는 취소 후 삭제할 수 있어요. |

### API 엔드포인트 (안)
```
DELETE /api/v1/me/images
Request: { "imageIds": ["img_1", "img_2"], "keepResults": true }
Response: { "success": true, "data": { "deletedCount": 2, "revokedShareLinks": 1 } }

DELETE /api/v1/me/data   → 전체 데이터 삭제 (탈퇴 플로우에서 재사용)
```

### Use Case 시나리오
**정상 시나리오:**
1. 상담이 끝난 사용자가 데이터 관리 화면에서 "원본 전체 삭제" 선택
2. "추천 결과는 남기기" 옵션 켠 채 삭제 확정
3. 원본 즉시 삭제 완료 안내, 보관함의 추천 결과는 그대로 유지

**예외 시나리오:**
1. 원본 포함 공유 링크가 존재 → 링크 회수 동반 안내 → 확인 후 삭제 진행
